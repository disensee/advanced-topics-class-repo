Introduction

In this project we are going to continue our exploration of modular code. You could actually think of a function as a module, it's a group of code that is meant to be used together. Remember that we created a 'module' for the image gallery by moving all the code into a function. We also added parameters to the function that allow the image gallery to be displayed inside of any tag/element on a page, and to allow the image gallery to display any set of pictures.

Take a look at the .js file that this page links to. When you first look at the code, you'll see something that might look strange to you (unless you are quite familiar with JavaScript). In JavaScript, you can nest functions within functions. Many other languages don't allow this. But it's actually a very powerful feature. It allows us to organize the code within our module into functions. And, as you'll see when you look at the code, we can invoke these nested functions from within the body of the main (outer) function.

As you might already know, there are lots of things that you can do with functions in JavaScript that you can't do in other languages.

Review the JavaScript code in this page and in simple-module.js and then answer the following questions.

Questions

  1. Can you access any of the variables in the module from JavaScript code in this page? 
    The variables in the module cannot be accessed from this page because they are encapsulated in
a function.
  2. Can you invoke the the initialize() or update() methods from JavaScript code in this page? Why not? 
    No. In order to invoke either of those methods they would have to be returned in an object.
  3. What is the scope of a local variable in JavaScript? 
    A local variable in JavaScript can only be accessed from within the function it is declared in (function scoping).
  4. How does the scope of a local variable in Java differ from the scope of a local variable in JavaScript (it might seem like a subtle difference)? 
    Local variables in Java can only be accessed in the block of code they are declared in (block scoping).
  5. Would you say that the variables and methods inside the module are 'public' or 'private'? 
    Private. The variables can only be accessed from within the function they are declared in.
  6. What happens to a local variable if you fail to use the 'var' keyword when you declare it? 
    If the 'var' keyword is not used the variable is declared as a global variable.
