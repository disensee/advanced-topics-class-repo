Questions

  1. Explain the return value of the getMonth() method of the Date API in JavaScript. Be sure to mention the data type that the method returns. 
    getMonth() returns an integer (0-11) that represents the month of a specified date object. January is 0 and December is 11.
  2. Explain the return value of the getDay() method of the Date API in JavaScript? 
    getDay() returns an integer (0-6) that represents the day of the week for a specified date object. Sunday is 0 and Saturday is 6.
  3. Explain how the monthName variable is initialized. 
    The monthName variable is initialized by creating an array of strings (named monthNames) that represents the names of the months. The array works in place of an object because months 
are zero indexed. From here, monthName is declared and initialized to the month name in the monthNames array by using getMonth() to get the month's index in the array.
  4. What is the purpose of the dayNames array? 
    The dayNames array serves the same purpose as the monthNames array. It allows us to use getDay() to retreive an index and use that index to get the name of the day.
  5. What are at least two changes you could make to the code for the format() function to make it more flexible? 
    Parameters could be passed in so that any date object could be formatted. Decomposing the method also makes it more flexible because you can return various formats
instead of being stuck with the one return format.
  6. What is decomposition in programming? 
    Decomposition in programming is the process of taking a complex problem and breaking it down into parts that are easier to undersatnd, program, and maintain.
  7. What is the single responsibility rule (SRR)? 
    The single responsibility rule states that every class or module in a program should be responsible for a single part of the program's functionality.
  8. As a programmer, do you need to understand all the code in the .js file in order to us it to display formatted dates in a web page (such as this one)? 
    No
  9. Explain the abstraction principle. 
    The abtraction principle's main goal is to hide complexity by hiding unncessary details from a user. This enables a user to implement more complex logic on top of 
the provided abstraction without having to understand or think about the hidden complexity.
