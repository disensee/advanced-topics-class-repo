Writing Modular Code

What is Modular Code?

We are going to turn our image gallery code into a module. Note that 'module' is a loaded term. Like many other terms in programming, it can mean different things in different languages. If you asked a few programmers what a module is, you'd probably get different answers. To make things even more confusing, JavaScript added a keyword/construct named 'module' just a few versions ago.

Here is the definition of 'modular programming' according to Wikipedia:

Modular programming is a software design technique that emphasizes separating the functionality of a programme into independent, interchangeable modules, such that each contains everything necessary to execute only one aspect of the desired functionality.

This sounds like a mouthfull, but hopefully by the time you complete this assignment, it will make more sense. Here's another definition of a module (according to webopedia.com):

A module is a part of a program. Programs are composed of one or more independently developed modules that are not combined until the program is linked.

As this course progresses, we will create many modules and eventually link them together to create working applications. Note that one of greatest things about modular code is that it can be re-used in different places without the need to duplicate code. Remember that you should avoid duplicating code! Duplicate code leads to maintenance nightmares and opens the door for buggy programs (we can discuss this in class if you are unclear about the problems that duplicate code presents).

Different programming languages have different approaches for creating modules. For example, in Java and C# you can organize your code into classes. Classes can be thought of as modules because they are used to organize your code into logical parts. Actually, you could argue that a function is a module because it organizes multiple lines of code that can be executed together. In the past, JavaScript did not support classes, but the latest version of JavaScript (ES6) now supports classes. So until very recently, JavaScript developers found ways to simulate classes by using functions (we'll explore some of these techniques throughout this course).

Note that if you search the web for info on modules and JavaScript you may get various explanations (because modern JavaScript has a construct called a 'module' which is not exactly the same as a class = CONFUSING!).

Code Review

  • First look at the image-gallery.js file (in the step02 folder) and note that we've created a function called ImageGallery. It takes two parameters, the first is the element from the HTML document that will contain the image gallery. The second param has the data that we wish to display (an array of images). These two params allow our module to have a lot of flexibility. Remember that in our original code (in step 1), the values in images array was hard-coded into our .js file, which made it difficult to reuse that .js file in different pages. We can now display any images, and we can also 'target' any tag on a page to be the container for our image gallery. The point is that passing parameters into functions is a great way to write flexible code!

  • Note that the window.addEventListener() method call has been removed from the .js file and moved into this page. Let's take a look at whats inside that bit of code on this page (look at the script tag at the top of this page). This page will wait until it finishes loading, and then initialize the image gallery by invoking the ImageGallery function.

  • Note that the css code has changed. The .css file has been renamed since it specifically contains styles related to the image gallery module. Instead of using ID selectors, we are now using class selectors. We did this so that you could potentially put two image galleries on the same page (remember that IDs should be unique).

  • Note that this module is a bit limited because we really have no way to interact with the image gallery after it has been initialized and injected into the page. For example, if wanted to add new images to the gallery after it has already been created, we don't have a good way to do it. Right now our module is sort of like 'set it and forget it'. But we'll soon create modules that we can interact with.

and the CSS/HTML could be pointing to multiple image galleries.
